Deliver OINK first-contact as a complete vertical slice (lift useInbox → ProfileTab UI/state machine → inbox refresh wiring → styles). After this plan, a logged-in user views any non-self profile, taps "OINK", and sees the button transition default → sending → sent (or default → sending → failed → retry). Output: Single `useInbox()` call at the App level; MessagesTab and ProfileTab receive what they need via props; ProfileTab implements the four-state OINK machine (`idle | sending | sent | failed`) with exact labels per D-09 / D-10.
</objective>

| Boundary | Description |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | Server response body for sendOink | accept | `sendOink` returns true only when the response HTML contains the exact server-rendered success string "Hey PiG your OINK! was sent.". The check is a string-includes inside the auth boundary; any tampering would require an active MITM, which is out of scope (HTTPS). |
| T-03-02 | Tampering / Double-submit | User tap-spamming OINK | mitigate | Handler guard `if (oinkStatus === 'sending' || oinkStatus === 'sent') return` prevents concurrent calls and prevents re-send after success. The `disabled` attribute also blocks at the UI layer. Failed state is intentionally re-tappable per D-10. |
| T-03-03 | Information Disclosure | OINK target user id in URL | accept | `data.id` is the canonical numeric id returned by the profile API; the user already has access to this id by virtue of viewing the profile. No new exposure. |
| T-03-04 | Repudiation | No client-side audit trail of OINKs sent | accept | Out of scope — bookmarklet does not persist state, and the server already logs all OINK attempts. |
| T-03-05 | Elevation of Privilege | Lifting useInbox to App | mitigate | Strictly a refactor — `useInbox` already runs in every Messages-tab mount today. Moving it to App means it runs once at mount instead of once per MessagesTab/ProfileTab mount. The not-logged-in guard inside `useInbox` remains the single point of auth detection and still drives the not-logged-in overlay in MessagesTab. |
| T-03-06 | XSS | Rendering profile fields and OINK labels | mitigate | All OINK labels are static literals defined in the component. Profile field rendering is unchanged (no `dangerouslySetInnerHTML` anywhere). Preact escapes interpolated text by default. |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies added; reuse of existing Preact + `sendOink` already in `src/api.ts`. Package legitimacy gate is N/A. |
</threat_model>
