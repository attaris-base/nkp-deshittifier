## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline element gaps |
| sm | 8px | Compose bar gaps, thread scroll padding, button gaps |
| md | 16px | Default horizontal padding for all list rows and sections |
| lg | 24px | Empty state padding top/bottom |
| xl | 32px | Major section breaks (empty state full padding) |
| 2xl | 48px | Loading state vertical padding |
| 3xl | 64px | Empty state vertical centering padding |

Exceptions (all values are multiples of 4):
- Search bar vertical padding: 8px (inner input) / 8px (container)
- Search bar horizontal padding: 16px input inner, 16px container outer
- Load more button vertical padding: 16px (matches md token)
- OINK button area: 16px padding (matches existing `.nkp-msg-btn-wrap`)

Accessibility constraint (not a spacing token): all interactive elements must meet a minimum touch target height of 44px. Enforce via `min-height: 44px` on the search input field — this is an accessibility floor, not a spacing value.

## Component Inventory

### New: Search Bar (MessagesTab inbox view)

**Class:** `.nkp-inbox-search`

**Primary visual anchor:** The thread list is the primary focal point of the inbox view. The search bar is a supporting element positioned above the fold — it is always visible but secondary to the content it filters.

Layout: sticky below the `.nkp-header`, above the thread list. Full-width container with `--bg-2` background and `border-bottom: 1px solid var(--border)`. Always visible — never collapsed.

```
.nkp-inbox-search {
  padding: 8px 16px;
  background: var(--bg-2);
  border-bottom: 1px solid var(--border);
  position: sticky;
  top: [header height ~41px];
  z-index: 4;
}
.nkp-inbox-search-input {
  width: 100%;
  background: var(--bg-card);
  border: 1px solid var(--border);
  border-radius: 20px;        /* pill — matches compose input */
  padding: 8px 16px;
  color: var(--text);
  font-size: 14px;
  font-family: inherit;
  outline: none;
  min-height: 44px;           /* accessibility touch target floor — not a spacing token */
}
.nkp-inbox-search-input::placeholder { color: var(--text-dim); }
.nkp-inbox-search-input:focus { border-color: var(--accent); }
```

States:
- **Default (empty):** border `var(--border)`, placeholder visible
- **Focused (empty):** border `var(--accent)`, placeholder still visible
- **Active (has value):** border `var(--accent)`, value text `var(--text)`
- **Loading (debounce in flight):** no visual change — the thread list below shows `Spinner` while results load
- **No results:** thread list replaced by empty state (see Copywriting)

Interaction rules:
- Debounce: 300ms after last keystroke before calling `searchMessages(query)`
- Minimum query length: 2 characters — below 2 chars, show full inbox (no search call)
- Clearing the field (empty string): revert to full inbox display immediately
- `type="search"` on the `<input>` — enables native clear button on iOS/Safari

### New: Load More Button (MessagesTab inbox view)

**Class:** `.nkp-inbox-load-more` (new, separate from existing `.nkp-load-more` on NearbyTab)

Placed inline at the bottom of the thread list, scrolls with content. Not sticky.

```
.nkp-inbox-load-more {
  width: 100%;
  padding: 16px 16px;
  background: transparent;
  border: none;
  border-top: 1px solid var(--border);
  color: var(--accent);
  font-size: 14px;
  font-weight: 700;
  cursor: pointer;
  font-family: inherit;
}
.nkp-inbox-load-more:disabled {
  color: var(--text-dim);
  cursor: not-allowed;
  opacity: 1;    /* do NOT reduce opacity — text color change alone signals exhausted */
}
```

States:
- **Default (more pages available):** accent text, pointer cursor — tappable
- **Loading (fetch in flight):** disabled, text "Loading…", no spinner (button-level feedback is sufficient)
- **Exhausted (`next_cursor` null/empty):** disabled, text "No more messages", `var(--text-dim)` text, `cursor: not-allowed` — button stays visible, does not disappear

### Modified: OINK Button (ProfileTab)

Reuses existing `.nkp-btn .nkp-btn-primary .nkp-btn-full` within `.nkp-msg-btn-wrap`.

Three states — controlled by local `useState` in `ProfileTab`:

| State | Label | Classes | Disabled | CSS notes |
|-------|-------|---------|----------|-----------|
| Default | `OINK` | `nkp-btn nkp-btn-primary nkp-btn-full` | false | Existing primary button — no changes needed |
| Success | `OINK sent ✓` | `nkp-btn nkp-btn-primary nkp-btn-full` | true | `opacity: 0.5; cursor: not-allowed` via `:disabled` — matches existing `.nkp-compose-send:disabled` pattern |
| Failure | `OINK failed — retry` | `nkp-btn nkp-btn-outline nkp-btn-full` | false | Outline variant — re-tappable |

The `.nkp-btn:disabled` rule needs to be added to `STYLES`:
```
.nkp-btn:disabled { opacity: 0.5; cursor: not-allowed; }
```

Note: The existing `.nkp-compose-send:disabled` rule handles the send button separately. The new rule applies to the generic `.nkp-btn` class.

## Interaction Contracts

### Search

| Trigger | Response |
|---------|----------|
| User types 1 char | No API call; full inbox remains |
| User types 2+ chars | Start 300ms debounce timer |
| Debounce fires | Call `searchMessages(query)`; show `Spinner` in thread list area |
| Results arrive | Replace thread list with search results; no empty state unless results array is empty |
| Results empty | Show empty state: "No results for [query]" |
| User clears field | Immediately restore full inbox (no API call) |
| Search is active | Load more button is hidden (pagination does not apply to search results) |

### Load More

| Trigger | Response |
|---------|----------|
| Tap (available) | Disable button, set text "Loading…", call `fetchMoreMail(next_cursor)`, append threads |
| Fetch succeeds | Re-enable button if new `next_cursor` returned; set exhausted state if not |
| Fetch fails | Re-enable button, show `ErrorBanner` above the button area |
| `next_cursor` null | Set exhausted state immediately (no tap needed) |

### OINK Button

| Trigger | Response |
|---------|----------|
| Tap (default) | Disable button immediately (prevent double-tap), call `sendOink(userID)` |
| `sendOink` returns `true` | Set button to success state ("OINK sent ✓", disabled); call `useInbox.refresh()` in background |
| `sendOink` returns `false` | Set button to failure state ("OINK failed — retry", re-tappable) |
| Tap (failure state) | Return to default state briefly, retry `sendOink` |

## Accessibility

| Element | Requirement |
|---------|-------------|
| Search input | `type="search"`, `aria-label="Search messages"`, `autocomplete="off"` |
| Load more button | `type="button"`, `aria-disabled="true"` when exhausted (matches visual disabled) |
| OINK button | `type="button"`, `aria-disabled="true"` in success state |
| OINK success state | `aria-live="polite"` region wrapping the button so screen reader announces state change |
| Load more loading | `aria-busy="true"` on button while fetching |

All `<button>` elements must have explicit `type="button"` — enforced by Biome `useButtonType`.
