<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user keyboard → search input → searchMessages URL | User-controlled string flows into URL query parameter |
| browser → nastykinkpigs.com | Auth via session cookie |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `searchMessages` URL query parameter | mitigate | Wrap `query` in `encodeURIComponent(query)` before interpolating into the URL. Without this, characters like `&`, `?`, `#`, or whitespace from the user's keystroke could inject additional params or break the request shape. |
| T-02-02 | Tampering / XSS | Rendering search result fields (`name`, `subject`, `preview`) | mitigate | Render all result fields through Preact's standard JSX text interpolation (`{r.name}`, `{r.preview}`) — Preact escapes by default. Do NOT use `dangerouslySetInnerHTML` anywhere. The adapter passes strings through unchanged into InboxView which uses `{t.sender_name}` etc. |
| T-02-03 | Information Disclosure | Search query in URL → server log | accept | The server already sees the user's queries on any search request; same as native site. The bookmarklet adds no new disclosure surface beyond what already exists when the user uses search on nastykinkpigs.com directly. |
| T-02-04 | Denial of Service | Rapid keystrokes flooding the server | mitigate | 300ms debounce inside the useEffect cleanup function ensures only the most recent query (after a typing pause) reaches the server. The clearTimeout in the effect cleanup function guarantees superseded queries never fire. |
| T-02-05 | Spoofing | `MessageSearch` shape assumption | mitigate | Return type assertion `return json as MessageSearch` is a trust-the-server pattern (consistent with existing api.ts). Out-of-band server changes would surface as TypeScript runtime undefined access; not a credible attack from within the auth boundary. |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies added; reuse of existing Preact + hand-rolled hooks/styles. Package legitimacy gate is N/A. |
</threat_model>
